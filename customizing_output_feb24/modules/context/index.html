Six context engineering exercises

Context engineering shapes the model’s working environment: what it prioritizes, ignores, trusts, remembers, and refuses. These six prompts isolate common levers used in real systems (tutors, RAG, evaluators).

Suggested flow: Start with priority + filtering (1–2), then audience (3), then state/history (4), then grounding (5), then authority (6).
Start: Example 1 → Jump to grounding (Ex. 5)

How to run as a lab

  • Run each prompt “as-is,” then change exactly one context lever.
  • Collect outputs and annotate: what changed, what stayed stable, what broke.
  • Require a short “context diff” note: what you edited and why.
