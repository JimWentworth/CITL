Example 5: Teach a custom labeling schema (taxonomy + edge cases)

Define categories and disambiguation rules; the model applies them to new items.

Try this prompt

Copy/paste into your LLM. Replace bracketed placeholders as needed.

You are a content triage assistant. Classify each item into exactly ONE label using the schema below.

Labels:
1) FACT: Verifiable statement about the world.
2) OPINION: Value judgment or preference.
3) INSTRUCTION: Tells someone what to do.
4) QUESTION: Seeks information.
5) META: About the conversation, the prompt, or the model.

Rules:
- If an item contains both a fact and an opinion, choose the dominant intent.
- Rhetorical questions that imply a judgment are OPINION unless they genuinely seek info.
- Statements about what "you (the assistant)" will do are META.

Examples:
"Paris is the capital of France." -> FACT
"Chocolate ice cream is the best." -> OPINION
"Click the Submit button." -> INSTRUCTION
"How do I reset my password?" -> QUESTION
"As an AI model, I cannot browse the web." -> META
"Isn't it obvious this policy is unfair?" -> OPINION

Now label the items below. Output as a numbered list: N) LABEL — short justification (max 12 words).

Items:
1. "The meeting starts at 3pm."
2. "Please summarize the attached document."
3. "I think this interface is confusing."
4. "Can you explain what 'latent space' means?"
5. "Earlier you said you would remember this."

What this teaches

  • Schema induction (lightweight ontology)
  • Edge-case control via explicit rules
  • Justification constraint to prevent rambling

Facilitator tip

Add one new edge-case rule and rerun; observe boundary changes.
