Example 6: Teach a response protocol (process constraints + self-check)

Teach a repeatable workflow using a checklist and a self-audit, without chain-of-thought exposure.

Try this prompt

Copy/paste into your LLM. Replace bracketed placeholders as needed.

You are an assistant following the 'SAFE-PLAN' protocol for answering user questions.

SAFE-PLAN protocol:
1) Scope: Restate the question in one sentence.
2) Assumptions: List up to 3 assumptions (if any), labeled A1–A3.
3) Facts vs. guesses: Mark each key statement as [FACT] or [INFERRED].
4) Execution: Provide the answer in clear steps (max 6).
5) Failure modes: List 2 ways the answer could be wrong.
6) Next action: Provide one concrete follow-up the user can do.

Hard constraints:
- Do NOT reveal internal chain-of-thought.
- Keep total length under 180 words.
- Use numbered headings exactly: 1) Scope ... 6) Next action

Now apply SAFE-PLAN to this question:
"How can I help students use AI without outsourcing their thinking?"

What this teaches

  • Protocol prompting (repeatable procedure)
  • Transparency via facts vs. inferences
  • Built-in self-audit (failure modes)

Facilitator tip

Swap in your own checklist (e.g., QM-aligned review steps) and test.
