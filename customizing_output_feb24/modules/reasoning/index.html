Six reasoning engineering exercises

These activities focus on controlling how the model reasons: strategy, decomposition, representation, attention, iteration, and evaluation. The goal is more stable, auditable outputs—especially for teaching and assessment.

Recommended sequence: 1 → 2 → 4 for control basics, then 3 for representation, then 5–6 for refinement + evaluation workflows.
Start: Example 1 → Jump to iteration (Ex. 5)

Example 1: Reasoning strategy switch (same task, different approach)

Teach that you can control the model’s approach (strategy), not just its answer.

Strategy A/B/C comparison

Example 2: Forced decomposition (prevent big jumps)

Improve reliability by splitting a complex task into ordered sub-tasks with constraints.

Decomposition Alignment

Example 3: Representation engineering (change the output structure)

Show that forcing a representation can improve clarity and reasoning quality.

Representation Clarity

Example 4: Attention constraints (focus only on X, ignore Y)

Constrain the model’s attention so it evaluates the right dimension (and stops grading ‘vibes’).

Attention Scope control

Example 5: Iterative refinement loop (draft → critique → revise)

Teach multi-pass workflows that reliably improve output quality.

Iteration Revision loop

Example 6: Evaluation engineering (rank answers using a rubric)

Train the model to judge quality explicitly, not just generate content.

Evaluation Rubric scoring

Simple scoring rubric for outputs

  • Auditability: Can you see why it concluded what it did?
  • Completeness: Did it address all required parts?
  • Constraint adherence: Did it follow the structure and limits?
  • Teaching usefulness: Does it support learning (not just answering)?
