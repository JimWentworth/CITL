Understanding Context Window Limits

Large language models can only “remember” a finite amount of information at one time. Just like our working memory, it has limits to the amount of information it can process. This boundary is called the context window. When you upload or paste more text than the model can fit into that window, it must ignore, compress, or summarize parts of it. This is where errors, omissions, and hallucinations begin to appear. 

In this activity, you’ll use a large document to intentionally exceed the context window in Copilot and observe the results.

Step 1 — Open a New Copilot Chat

Start fresh so the chat contains no previous context. Then upload this PDF on Energy and AI. You’ll see the file appear at the top of the chat window.     

Note: Uploading a large file does not mean Copilot “reads” the entire document. It processes only what can fit inside its available context window. In this case it may not be able to load at all.

Step 2 — Ask for a Summary

Ask Copilot: “Summarize the entire document in 8–10 bullet points.” If it can provide a summary, it will seem coherent but will likely miss sections or compress them too aggressively.

Step 3 — Ask About Details Near the End

Now ask a question about information from the final chapters—for example:

“What were the central inputs used to inform the model?”

Because Copilot’s context window is exceeded, it may:

• provide no response
• fabricate content that “sounds right”
• ignore the question and give a general answer
• respond with uncertainty about missing content

Conceptual Wrap-Up

Working with a long PDF highlights the difference between “uploaded data” and “active context.” Models can't always load an entire document into memory. When a file is too large, much of it never becomes available for direct reasoning. This is why summaries can seem coherent but fail to reflect the actual content of the final pages. The context window differs significantly between different platforms. The Tool Comparison Matrix highlights these differences.  

Show Detailed Explanation

The file upload feature can give the impression that the model has absorbed the entire document, but what actually happens is more constrained and dependent on the document length. The model may extract text in chunks and pull segments into the context window as needed. When the document is extremely large, only a fraction is accessible at any given moment. Attempts to refer to the end of the document may pull in unrelated fragments or cause the model to guess based on its general knowledge rather than the specific content of the file.

Trying contradictory questions is an effective diagnostic tool. If you deliberately ask about claims the document doesn’t support and Copilot still provides confident answers, you have exceeded the model’s ability to ground its responses in the actual file. These failures are not random—they reveal the fundamental limits of the model’s internal memory.

Understanding these limitations sets the stage for Retrieval-Augmented Generation (RAG) workflows in NotebookLM and other tools, which we will explore next. RAG systems work around context limits by indexing, chunking, and retrieving specific fragments rather than trying to load the full document into a single conversation window.
