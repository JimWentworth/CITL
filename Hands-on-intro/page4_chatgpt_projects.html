ChatGPT Projects & Collaborative Workflows

ChatGPT Projects allow you to build and share a persistent workspace that holds documents, examples, prompts, and notes. Unlike NotebookLM, which stays strictly grounded in the uploaded sources, ChatGPT blends your documents with its full model knowledge unless you instruct it otherwise. 

In this activity, you’ll join a shared project seeded with materials and experiment with workflows that support collaboration and course design.

Step 1 — Open the Shared Project

Use the link below to access the shared project in ChatGPT (it is best to login first). Inside the project you will see an environment containing previous chats from earlier versions of this workshop

This is a collaborative workspace. Any chats you have within this project are visible for all participants. That does not mean that the system automatically reviews prior chats for context. Each chat in a project starts fresh it doesn’t automatically remember what happened in other threads. You can also delete or archive your chats at any time.

Note: Review the chat list for chats dated January 9th to see extensive examples used to develop this workshop.

A major benefit of project folders is they can include custom instructions that apply to all chats within the project. This allows you to standardize project requirements like 'use University of Illinois branding when creating HTML pages' or 'write for an academic audience who prefer concise language not bullet points' These instructions persist for every chat in the project.

Projects also allow you to upload files to be used for retrieval augmented generation. A well defined set of standards and even code snippets can all be included at the project level to be referenced and re-used for all users in each new chat. This makes project folders ideal for things like course development where you want the standardization of documents for course design but also the power the of the LLM to help generate course content and ideas to build engagement.

Step 2 — Ask Questions to Reveal the RAG documents

In the project chat window, ask a question so you can understand the uploaded documents.

“Outline the documents used as RAG in this project folder”

ChatGPT should review and summarize the intention of the documents. You can also click on the add files icon near the title of the project to see uploaded files. 

These documents are all saved as markdown (.md) which helps the computer access content more efficiently. A well-organized project will also include a readme file for humans to understand the context of the project and documents. 

Step 3 — Generate a Draft Asset (Syllabus, Module, or Summary)

Ask ChatGPT to generate an outline, module overview, or assignment description using the project documents. Try a prompt like:

“Using the RAG style guide and the course outline, create a 2-week module on AI literacy.”

Before responding ChatGPT will review the RAG documents for information related to the prompt and then use that as additional context for the LLM to form it's response. It's using your documents in combination with the large language model. This is different from NotebookLM, which answers only from the sources you uploaded without accessing additional sources from the larger LLM.

Note: The markdown files used as RAG for this example were created to support the development of CITL's Certificate in AI Fluency.

Step 4 — Compare ChatGPT Projects to NotebookLM

Ask ChatGPT a question that requires inference beyond the uploaded files—for example:

“How will generative AI reshape the workforce over the next decade?”

ChatGPT will generate answers by merging your documents with external knowledge. NotebookLM would not answer this unless the information appears in the uploaded sources.

Conceptual Wrap-Up

ChatGPT Projects operate as a hybrid workspace: they maintain a persistent library of uploaded documents but remain connected to the model’s broader knowledge base. This makes Projects ideal for workflows that require both grounded references and creative synthesis, such as curriculum design, policy drafting, and collaborative editing.

NotebookLM is a closed system. It only uses the documents you upload and nothing more. This makes it ideal for accuracy, citation alignment, and document fidelity. It excels in academic reading, study guides, and student-facing exploration.

ChatGPT Projects operate differently. They store documents but draw on the full model unless you limit them. This hybrid behavior allows for broader creativity and synthesis—but requires clearer instruction when accuracy or citation alignment matters. You decide which mode is appropriate by how you frame your prompts.

Projects also enable genuine collaboration: multiple instructors can iteratively draft, revise, and expand materials. Files can be replaced or updated. Prompts can reference each other. The workspace evolves over time—something neither Copilot nor NotebookLM currently supports in the same way.
