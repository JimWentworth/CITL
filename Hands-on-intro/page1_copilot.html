Effective Prompting

In this activity, you will iterate on a simple prompt using a new Copilot chat for every step so you can see how each added instruction reshapes the model’s output. This isolates each prompt and prevents context from earlier chats influencing the results of the next chat. 

Step 1 — Base Prompt

Start with the simplest possible request. Open a new Copilot chat and paste this prompt:

Explain generative AI .

Step 2 — Add a Role

Now open a new Copilot chat again (don't append this to the first chat) . Adding a role guides tone, perspective, and domain emphasis.

Act as a third grade teacher and explain generative AI.

Step 3 — Add an Audience

Open a new Copilot chat again. Audience guides vocabulary, complexity, and pacing.

Act as a third grade teacher and explain generative AI to your colleagues.

Step 4 — Add Constraints

Open another new Copilot chat. Constraints reduce drift, enforce readability, and limit hallucination.

Act as a college instructor. Your audience is first-year undergraduates. Explain generative AI. Apply these constraints: Use plain language, stay under 500 words, avoid metaphors.

Step 5 — Add Output Structure

Open a new Copilot chat again. Structure shapes the response into a predictable, skimmable format.

Act as a college instructor. Your audience is first-year undergraduates. Explain generative AI. Apply these constraints: Use plain language, stay under 150 words, avoid metaphors. Use this structure: a 2-sentence definition, 3 concise bullet points, and one real-world example.

Conceptual Wrap-Up

As you interact with Copilot, you’ll see each chat appear as a separate entry in the chat history on the left side of the interface. Copilot treats each of these threads as its own workspace with its own context. Within an individual chat every message adds to the assumptions the model uses to shape subsequent responses.

This separation is helpful when refining a single idea because staying in one chat allows Copilot to carry forward tone, depth, and structure without you restating your preferences. But it becomes limiting when you change direction. Even if your intentions shift, the ongoing chat still carries earlier instructions and patterns. This is why starting a new chat gives you a cleaner context and a new entry in the sidebar—visually and conceptually separating tasks.

In this activity, you started a new chat at every step so you could isolate the effect of each added instruction—role, audience, constraints, and structure. Doing this prevented earlier messages from influencing later outputs and made it easier to observe how each layer reshaped the model’s behavior. Everyone's results will vary somewhat, but hopefully you saw how you could strategically improve prompts by providing additional context.

Show Detailed Explanation

Each chat on the sidebar represents a distinct context container. When you re-open a previous chat, Copilot loads the entire conversation history from that specific thread, effectively reconstructing all instructions you’ve given earlier. Inside a chat, the model attempts to remain consistent with the tone, depth, formatting, and intent you established in previous messages.

This continuity is powerful when you are revising or expanding a single task because the model can maintain coherence without needing repeated instruction. However, if you pivot to a new subject or try to compare different approaches, the accumulated assumptions become an obstacle. Copilot still thinks it should follow the patterns you set earlier—even though you’ve moved on.

Starting a new chat creates a new sidebar entry and wipes away all previous instructions. This allows you to test how role, audience, constraints, and structure independently shape a prompt. By comparing the results across these fresh chats, you saw how deliberate prompting changes output—transforming a generic explanation into something tailored, structured, and aligned with your teaching goals. Understanding this foundation will support everything you build later, including context window management, RAG workflows, multimodal prompts, and collaborative project-based interactions.

Many of these systems also contain a built in memory feature that captures your preferences over time. This is useful when you want the tool to remember context that's not likely to change regularly like 'I work at the University of Illinois'. Check the settings of your tool to review any saved memories and enable or disable this feature.
